# Equipment module

Core equipment inventory: CRUD, search/filter/pagination, department- and
status-scoped views, photo/manual uploads, and QR code generation. Registration
is normally done through the [`receiving`](../receiving/README.md) module
(which forces `status: PENDING_INSTALLATION`), but `POST /equipment` is
available for direct creation too.

## Schema (`Equipment`)

| Field | Type | Notes |
|-------|------|-------|
| `assetNumber` | string | unique; auto-generated (`EQ-<year>-<5-digit sequence>`) if omitted |
| `name` | string | required |
| `category` | string | required |
| `manufacturer` | string | required |
| `model` | string | optional |
| `serialNumber` | string | unique, required |
| `department` | ObjectId → `Department` | required |
| `roomLocation` | string | optional |
| `supplier` | string | optional |
| `purchaseDate`, `installationDate`, `warrantyStartDate`, `warrantyEndDate` | Date | optional |
| `installedBy` | ObjectId → `User` | set by `receiving` on installation confirmation |
| `cost` | number | optional |
| `status` | `EquipmentStatus` | default `PENDING_INSTALLATION` — see [lifecycle](../receiving/README.md#state-machine) |
| `photoUrls`, `manualUrls` | string[] | default `[]` |
| `qrCodeUrl` | string | auto-generated on create |
| `pmFrequencyDays` | number | default `90` |
| `calibrationFrequencyDays` | number | default `365` |
| Virtual `isWarrantyExpired` | boolean | computed from `warrantyEndDate`, not stored |
| `createdBy`, `updatedBy`, `isDeleted`, `deletedAt`, `createdAt`, `updatedAt` | — | from `BaseSchema` |

`EquipmentStatus`: `PENDING_INSTALLATION` \| `WORKING` \| `UNDER_REPAIR` \| `CONDEMNED` \| `DECOMMISSIONED`

## Endpoints

| Method | Path | Roles | Notes |
|--------|------|-------|-------|
| `POST` | `/equipment` | `ADMINISTRATOR`, `STORE_OFFICER`, `BIOMEDICAL_ENGINEER` | Auto-generates asset number + QR code if not supplied |
| `GET` | `/equipment` | any authenticated (department-scoped) | Paginated, filterable list |
| `GET` | `/equipment/department/:departmentId` | any authenticated (guarded + department-scoped) | Excludes `PENDING_INSTALLATION` — "active inventory" view |
| `GET` | `/equipment/status/:status` | any authenticated (department-scoped) | |
| `GET` | `/equipment/:id` | any authenticated | Department populated |
| `PATCH` | `/equipment/:id` | `ADMINISTRATOR`, `BIOMEDICAL_ENGINEER` | |
| `DELETE` | `/equipment/:id` | `ADMINISTRATOR` | Soft delete, `204 No Content` |
| `POST` | `/equipment/:id/photos` | `ADMINISTRATOR`, `STORE_OFFICER`, `BIOMEDICAL_ENGINEER` | Multipart field `photos`, up to 10 images |
| `POST` | `/equipment/:id/manual` | `ADMINISTRATOR`, `STORE_OFFICER`, `BIOMEDICAL_ENGINEER` | Multipart field `manual`, single file (image or PDF) |
| `POST` | `/equipment/:id/qr-code/regenerate` | `ADMINISTRATOR`, `BIOMEDICAL_ENGINEER` | Re-encodes `{assetNumber}:{equipmentId}` |
| `GET` | `/equipment/:id/qr-code` | any authenticated | `{ qrCodeUrl }` |

### `GET /equipment` query params (`QueryEquipmentDto`)

Extends the standard `page`, `limit`, `sort`, `search` (matched against a
text index on `name`/`category`/`manufacturer`) with:

| Param | Type | Notes |
|-------|------|-------|
| `department` | MongoId | filter by department |
| `status` | `EquipmentStatus` | exact match |
| `category` | string | case-insensitive regex |

## Sample requests

**Create equipment**

```http
POST /equipment
Authorization: Bearer <accessToken>
Content-Type: application/json

{
  "name": "Patient Monitor",
  "category": "Diagnostic Imaging",
  "manufacturer": "Philips",
  "model": "IntelliVue MX450",
  "serialNumber": "SN-00012345",
  "department": "665f1b2c8a1e2f0012345678",
  "pmFrequencyDays": 90
}
```

```json
{
  "data": {
    "id": "665f1b2c8a1e2f001234567b",
    "assetNumber": "EQ-2026-00042",
    "name": "Patient Monitor",
    "status": "PENDING_INSTALLATION",
    "qrCodeUrl": "http://localhost:3000/uploads/qr-codes/....png",
    "...": "..."
  },
  "meta": {}
}
```

**Search + filter**

```http
GET /equipment?department=665f1b2c8a1e2f0012345678&status=WORKING&search=monitor&page=1&limit=20
Authorization: Bearer <accessToken>
```

**Upload photos**

```http
POST /equipment/665f1b2c8a1e2f001234567b/photos
Authorization: Bearer <accessToken>
Content-Type: multipart/form-data

photos: <file1.jpg>
photos: <file2.jpg>
```

## Business logic notes

- Asset numbers are generated by a `Counter`-backed `SequenceService` for
  gap-free, race-safe sequential numbering per year.
- QR payload encodes `{assetNumber}:{equipmentId}`; the PNG is stored via the
  active [`files`](../files/README.md) storage provider under `qr-codes/`.
- `assetNumber` and `serialNumber` are both unique — duplicate values return
  `409`/`400` depending on the Mongo error mapping in the global filter.
- `findWarrantyExpiringWithinDays(days)` backs the notifications cron job's
  warranty-expiry check.
- Department scoping: `ADMINISTRATOR`/`STORE_OFFICER` see all equipment;
  `BIOMEDICAL_ENGINEER`/`DEPARTMENT_USER` only see equipment in their
  assigned departments (empty result set if none are assigned).
